# 02: Required User Information During Registration

## Meta

Status
: Decided

Updated
: 16-Jun-2026

## Problem Statement

The platform connects vulnerable people in need of assistance with volunteer helpers. Therefore, the team had to decide which personal information users must provide during registration.
Collecting too little information would make it difficult to verify users and could increase the risk of misuse. Collecting too much information, however, could discourage users from registering and raise privacy concerns.
The challenge was to find a balance between security, trust and usability.

## Decision

The team decided that all users must provide the following information during registration:
- First Name
- Last Name
- Email Address
- Phone Number
- Address
- Postal Code
- Date of Birth
- Password

This information provides a sufficient level of identification while supporting trust and accountability between users.

The decision was made because the platform is intended for real-life interactions between helpers and people in need of assistance. Therefore, user verification and misuse prevention are important requirements.

Decision was taken by: The project team

## Regarded Options

### Option 1: Minimal Registration

Required information:

- Name
- Email Address
- Password

# Pros
- Fast registration process
- Low entry barrier
- Easy implementation

# Cons
- Limited user identification
- Higher risk of fake profiles
- Lower level of trust

### Option 2: Standard Registration

Required information:
- First Name
- Last Name
- Email Address
- Phone Number
- Password

# Pros
- Better user identification
- Moderate registration effort
- Improved trust

# Cons
- Still limited information for verification
- Less suitable for handling misuse cases

### Option 3: Extended Registration (Chosen Option)

Required information:

- First Name
- Last Name
- Email Address
- Phone Number
- Address
- Postal Code
- Date of Birth
- Password

# Pros
- Stronger user identification
- Increased trust and accountability
- Better support for misuse prevention
- More suitable for real-world interactions

# Cons
- Longer registration process
- More personal data must be stored

The regarded options were discussed within the project team during the requirements analysis phase.

The team compared the different registration approaches based on the following criteria:

- User safety
- Trust between users
- Risk of misuse
- Registration effort

No separate prototype implementation was created because the decision was made before the development phase.
