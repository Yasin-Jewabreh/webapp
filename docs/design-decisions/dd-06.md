# [06: Seperate Übersicht angenommener Aufträge und offene Aufträge]

## Meta

Status
: **Work in progress** - 

Updated
: 23-Jun-2026

## Problem Statement

Helfer können in der Anwendung offene Aufträge ansehen und einen passenden Auftrag annehmen. Nachdem der Button "Annehmen" gedrückt wurde wird man auf eine Seite weitergeleitet die dir den Auftrag nochmal bestätigt. Danach kann man den Button "Zurück zur Übersicht" auswählen womit man auf die angenommene Auftragsübersicht geleitet wird. 

Denn es wäre für den Helfer zu anstrengend wenn man sich das jedesmal selber merken müsste welche Aufträge man angenommen hat, wie der Hilfe suchende hieß, sein Alter und was seine benötigte Hilfe war. Deswegen soll anhand der 'nutzer.id' dem Helfer die eigenen angenommenen Aufträge aufzeigen. Und über die Ansicht kann der Helfer in dem er das Button 'Chatten' neben dem AUftrag auch direkt mit dem Hilfe suchenden Chatten. 

## Decision

Ich habe mich dafür entschieden nochmal eine seperate route: '/helfer/meine-auftraege' zu erstellen die auch mit dem '@login_required' geschützt ist damit man nur anhand des eingeloggten nutzer.id weiß welcher nutzer gerade eingeloggt ist. Außerdem werden natürlich auch nur die Aufträge angezeigt die auch wirklich mit 'angenommen=True' deklariert wurden. Für den Layout und Design wurde nochmals eine seperate meine_auftraege.html erstellt.

## Regarded Options

Option 1: Man würde alle angenommenen so wie auch offenen Aufträge in einer Übersicht angezeigt. 
-> Dies würde dazu führen das es nach einer kurzen Zeit mit nur schon ein Dutzend Aufträgen man keine Übersicht mehr über die Aufträge haben würde welche offen sind und welche angenommen wurden.

Option 2: Man erstellt zwei seperate Routings damit eine klare Trennung zwischen offene und angenommene Aufträge erzielt wird.
-> Damit wird auch anhand der 'nutzer.id' auch nur die angenommenen Aufträge angezeigt die man selber angenommen hat. Es gibt eine viel bessere Trennung und Übersicht über die Aufträge. Es ist auch viel einfacher damit die Funktion Chat direkt mit dem Hilfe suchenden einzubinden.
